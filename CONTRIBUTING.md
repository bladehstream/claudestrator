# Contributing to Claudestrator

Thank you for your interest in contributing to Claudestrator!

## Ways to Contribute

### 1. Add New Skills

The most valuable contributions are new skill definitions for different domains:

**High-value skill areas:**
- Frontend frameworks (React, Vue, Angular)
- Backend frameworks (Django, Rails, Spring)
- Mobile development (React Native, Flutter)
- Data science (Python, Jupyter, pandas)
- DevOps (Docker, Kubernetes, CI/CD)
- Database design (SQL, NoSQL)

**To add a skill:**
1. Copy `skills/skill_template.md` to the appropriate directory
2. Fill in all metadata fields (crucial for matching)
3. Write comprehensive patterns and examples
4. Add entry to `skills/skill_manifest.md`
5. Submit PR with example use case

### 2. Improve Existing Skills

- Add more code patterns and examples
- Expand keyword lists for better matching
- Improve anti-pattern documentation
- Add domain-specific quality standards

### 3. Enhance the Protocol

- Improve skill matching algorithm
- Add new task types
- Enhance journal format
- Better context efficiency strategies

### 4. Documentation

- Usage examples for different project types
- Troubleshooting guides
- Integration tutorials

## Skill Quality Guidelines

Good skills should have:

- **Complete metadata**: All frontmatter fields filled accurately
- **Clear role description**: What the agent does and when to use it
- **Concrete patterns**: Code examples, not just descriptions
- **Quality standards**: What "good" looks like
- **Anti-patterns**: Common mistakes to avoid
- **Output expectations**: Checklist of what agent should produce

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/new-skill`)
3. Make your changes
4. Test with Claude Code if possible
5. Submit PR with clear description

## Code of Conduct

- Be respectful and constructive
- Focus on improving the framework
- Share knowledge generously

## Questions?

Open an issue for discussion before major changes.
