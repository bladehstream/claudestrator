# /skills - Show Loaded Skills

Display all skills currently loaded and available for task matching.

## Display Format

```
═══════════════════════════════════════════════════════════
SKILL LIBRARY
═══════════════════════════════════════════════════════════

Directory: [skill directory path]
Total skills: [count]

IMPLEMENTATION
  • html5_canvas    Canvas/game development
  • game_feel       Polish, effects, "juice"
  • react_components React component development

DESIGN
  • game_designer   Game mechanics, balance
  • api_designer    REST/GraphQL API design

QUALITY
  • qa_agent        Testing, verification
  • security_reviewer Security audits
  • user_persona    UX review, accessibility

SUPPORT
  • svg_asset_gen   SVG graphics creation
  • refactoring     Code restructuring
  • documentation   Technical writing

═══════════════════════════════════════════════════════════
Add skills: Drop .md files with valid frontmatter into
the skill directory. They'll be loaded next session.
═══════════════════════════════════════════════════════════
```

## Skill Details

To see details on a specific skill, user can ask:
"Tell me about the qa_agent skill" or "What does html5_canvas do?"

Then read and summarize the skill file:
- Role description
- Core competencies
- Task types it handles
- Keywords it matches
